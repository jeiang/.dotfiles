# Isolate Hermes from publication authority

Hermes runs as an unprivileged service on legion-node3 and prepares only local `codex/` worktrees it owns outright. A separate Telegram-approved Publication Broker owns the narrow GitHub write credential, keeps private bare mirrors that both feed Hermes repository content and isolate every credentialed git operation from Hermes-writable repository configuration, and publishes only a commit pinned by SHA in the approved request. The private Markdown Knowledge Base remains the authority for durable facts and Observed Snapshots remain time-bound evidence; snapshot collection itself stays off until Hermes is enabled. The service stays disabled until the persistent Volume, encrypted credentials, and OAuth enrollment are all complete (`docs/runbooks/hermes-activation.md`). No raw recurring snapshot is committed to the Knowledge Base.
