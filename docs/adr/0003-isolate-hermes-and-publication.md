# Isolate Hermes from publication authority

Hermes runs as an unprivileged service on legion-node3 and prepares only local `codex/` worktrees. A separate Telegram-approved Publication Broker owns the narrow GitHub write credential, while the private Markdown Knowledge Base remains the authority for durable facts and Observed Snapshots remain time-bound evidence. The service stays disabled until the node3 capacity gate, persistent Volume, encrypted credentials, and OAuth enrollment are all complete. No raw recurring snapshot is committed to the Knowledge Base.
